# Evaluation Specification

Benchmark details, scoring rules, and runner specification for the `evals/` harness.
For latest results, see [`README.md`](./README.md).

## Evaluation Policy: as-released best

Prioritizing **production-deployment assumptions** over strict fairness.

- Compare models using each model's officially recommended settings + publicly available acceleration (EAGLE, etc.)
- Launch configs are git-managed in `scripts/sglang-*.sh` for transparency
- Configuration differences are explicitly noted (e.g., Kimi K2.6 Phase 4 measurement was performed without speculative decoding because the EAGLE3 draft was not yet published. As of 2026-05, `lightseekorg/kimi-k2.6-eagle3` has been published and is now enabled)
- temperature=0, max_tokens=32768, N=1. MoE has minor fluctuations, but differences of 5% or more are conclusive
- Chat template / prompt format follows each model's official specification

## Benchmark List

| Benchmark | Test Content | Format | Size |
|---|---|---|---|
| **llm-jp-eval** (abbreviated) | Japanese general (common sense, QA, math) | exact_match / char_f1 / numerical match | 4 tasks (~6K questions) |
| **IgakuQA** (2018-2022) | JNMLE 5-year span | Point allocation (built-in `points` per question) + accuracy | 2000 questions / 2485 pts |
| **IgakuQA119** (119th) | 119th JNMLE (administered Feb 2025) | 500 pts max (compulsory 3 pts) / Overall / No-Img | 400 questions / 500 pts |
| **JMLE2026** (120th) | Latest JNMLE (administered Feb 2026). Completely absent from training data | 500 pts max (compulsory 3 pts) / Overall / Text-only | 400 questions / 500 pts |
| **JMED-LLM** (MCQ 3) | Medical QA, case-report ADE classification, radiology report TNM staging | `kappa(accuracy)`, CRADE uses linear-weighted | 3 tasks (~3K rows) |

Priority: **JMLE2026** (most recent, completely leak-free) > **IgakuQA119** (low contamination risk) > **JMED-LLM** (covers practical clinical tasks) > llm-jp-eval (Japanese general capability baseline) > IgakuQA (year-over-year trend confirmation).

## Benchmark Details

### llm-jp-eval (abbreviated)

Only 4 tasks generated by the preprocessing scripts from [`llm-jp/llm-jp-eval`](https://github.com/llm-jp/llm-jp-eval). Each task's metric is specified in the dataset JSON (`exact_match`, `char_f1`, `mathematical_equivalence`).

- JCommonsenseQA (common-sense MCQ, 5-choice)
- JEMHopQA (multi-hop QA, free-form)
- JSQuAD (extractive QA, free-form)
- MGSM-ja (math computation, numerical)

Preprocessing commands:

```bash
cd evals/datasets/llm_jp_eval && uv sync
for t in jcommonsenseqa jemhopqa jsquad mgsm; do
  uv run python scripts/preprocess_dataset.py -d "$t" -o ./dataset
done
```

HF authentication is not required (JCQA/JEMHopQA/JSQuAD are from GitHub raw; MGSM-ja is from HF but open). `uv run hf auth login` is only needed when adding HLE-family tasks in the future.

### IgakuQA (2018-2022)

[`jungokasai/IgakuQA`](https://github.com/jungokasai/IgakuQA) -- JNMLE 5-year span. Each question contains a `points` field (e.g., compulsory questions are 3 points), used directly as `points_possible`.

**Image-bearing questions**: The original repository contains only question text without the actual images. The runner makes the model **solve image-bearing questions as blind solving (text-only)** (same scope as PFN HF card / Kasai+ 2023 published LB = 2485 pts max).

Per-exam denominators:

| Year | Exam | Total (questions / pts) |
|---|---|---|
| 2018 | 112th | 400 / 499 |
| 2019 | 113th | 400 / 496 |
| 2020 | 114th | 400 / 496 |
| 2021 | 115th | 400 / 500 |
| 2022 | 116th | 400 / 494 |
| **Total** | 5-year aggregate | **2000 / 2485** |

The number of compulsory/general questions and their point allocation varies slightly per exam, so totals are not exactly 500 but range from 494 to 500 (using the question-side `points` as-is). `leaderboard` contains 5-year aggregates; `leaderboard_by_year` contains per-year results. Output JSONs also retain the `has_image` flag, so the `no_image` bucket (1471 questions / 1864 pts) is available from `leaderboard.no_image` if needed.

**Block-level breakdown**:

### JMED-LLM (MCQ 3 tasks)

3 tasks adopted from [`sociocom/JMED-LLM`](https://github.com/sociocom/JMED-LLM):

- **JMMLU-Med**: Medical MCQ (5 subjects: professional_medicine, medical_genetics, clinical_knowledge, anatomy, college_medicine)
- **CRADE**: Classify likelihood of ADE (adverse drug events) from case reports on a 4-level scale. **Ordinal with linear-weighted kappa**
- **RRTNM**: Predict TNM staging from radiology reports

Evaluation format follows the official LB: `kappa(accuracy)`. CRADE uses ordinal linear-weighted kappa; JMMLU-Med / RRTNM use standard Cohen's kappa.

The `kappa` calculation is implemented in `evals/tasks/jmed_llm/run.py:cohen_kappa` (no numpy, stdlib only). OOV (output does not match any choice) is counted as equivalent to the most distant category.

## Scoring Details

### Difference Between Score and Acc.

| | Calculation |
|---|---|
| **Acc.** (Accuracy) | Correct count / total questions. All questions weighted equally |
| **Score** | Points earned / maximum points. **Compulsory questions count as 3 points** (IgakuQA119 B/E blocks Q26-50) |

The JNMLE weights compulsory questions heavily, with a separate minimum threshold (must score 80%+ on compulsory). Score is effectively a "compulsory-weighted composite score"; Acc. is the "raw accuracy rate". Even with the same number of correct answers, missing many compulsory questions causes **Score to drop significantly more**. This is why both appear side by side on leaderboards.

Example: Gemini-2.5-Pro Overall is 485/500 (97.00%) / 389/400 (97.25%). Missed 11 out of 400 questions but lost 15 points in scoring -- reverse-calculating: 2 compulsory 3-point questions + 9 general 1-point questions missed (2x3 + 9x1 = 15).

### Vision Auto-Probe (automatic detection of image-bearing question support)

The `igakuqa119` / `jmle2026` runner sends a **synthetic red square PNG at startup and asks what color it is**, checking whether the response contains `red`. The `reasoning_content` is also included in the inspection, with `max_tokens=512` to give reasoning models sufficient headroom.

When vision is NG (not supported), **all benchmarks run in blind mode** (image-bearing questions are answered from text only). This populates the Overall column for text-only models as well, aligning them with the same scope as the published LB:

| Benchmark | Behavior when vision NG | LB-side rationale |
|---|---|---|
| `igakuqa` (2018-2022) | Always blind (image files not bundled) | PFN/Kasai+ also use text-only blind |
| `igakuqa119` | blind -> all 400 questions (Overall column populated) | LB entries like Preferred-MedLLM-Qwen-72B (text-only) appear in the Overall column (332/500, etc.) -> blind is the default |
| `jmle2026` | blind -> all 400 questions (Overall column populated) | LB default is `image_mode=blind`; text-only Qwen3.5 family also appears in the All (400) column |

`--no-vision` force-skips auto-probe -> all benchmarks run in blind mode. `jsamples[i].has_image` identifies image-bearing questions; the text-only subset is also recorded in `leaderboard.no_image`.

### Published LB Parallel Format

Each task is aggregated in the same column format as the official leaderboard and stored in the result JSON's `leaderboard` key. The `summarize.py` output includes a **leaderboard rows** section with Markdown rows that can be pasted directly into the official repository README.

| Benchmark | Format | Example |
|---|---|---|
| IgakuQA / IgakuQA119 | `Overall Score` / `Overall Acc.` / `No-Img Score` / `No-Img Acc.` | `461/500 (92.20%)` |
| JMLE2026 | `Overall Score` / `Overall Acc.` / `Text-only Score` / `Text-only Acc.` | `486/500 (97.20%)` |
| JMED-LLM | `kappa(accuracy)`, CRADE uses linear-weighted kappa | `0.54(0.53)` |
| llm-jp-eval | exact_match / char_f1 / mathematical_equivalence | `0.823` |

## Measurement Items

### Speed

Recorded per request:

- **TTFT** (Time To First Token): Prompt sent -> first token received
- **TTAT** (Time To Answer Token): Prompt sent -> answer start immediately after `</think>` (perceived latency during reasoning)
- **total_time_ms** / **prompt_tokens** / **reasoning_tokens** / **answer_tokens**
- **decode tok/s**: `(reasoning + answer) / (total - ttft)` computed post-hoc by `summarize.py`

Concurrent throughput (simultaneous 4/8/16 users) is not measured by this harness. Single-client only.

### Thinking Volume

- `reasoning_tokens` median and p90 (per task)
- `reasoning_tokens / answer_tokens` ratio
- accuracy vs `reasoning_tokens` trade-off (at equal accuracy, shorter thinking is operationally preferable)

Retrieved via SGLang's `--reasoning-parser` through OpenAI-compatible API's `reasoning_content` / `content` and `usage.reasoning_tokens`.

## Harness Usage Details

### Common Task Runner Flags

```
--base-url    SGLang endpoint (default http://localhost:8000)
--model       served-model-name
--task        Task name or "all" (depends on task family)
--output-dir  Result JSON output directory
--limit N     First N questions only (for smoke testing)
--no-think    chat_template_kwargs.enable_thinking=False
--max-tokens  default 32768
--temperature default 0.0
```

Task-family-specific flags:

| Family | Specific Flags |
|---|---|
| `tasks.llm_jp_eval_subset` | `--task {jcommonsenseqa,jemhopqa,jsquad,mgsm,all}` |
| `tasks.igakuqa` | `--years 2018 2019 ...` |
| `tasks.igakuqa119` | `--blocks 119A 119B ...` `--no-vision` (force-skip auto-probe) |
| `tasks.jmle2026` | `--blocks A B ...` `--no-vision` (force-skip auto-probe) |
| `tasks.jmed_llm` | `--task {jmmlu_med,crade,rrtnm,smdis,jcsts,all}` (`all` excludes smdis/jcsts; only runs when explicitly specified) |

Output: `<output-dir>/<task>.json` -- metrics, timing distribution, token distribution, `leaderboard`, all sample raw/extracted/correct-or-not, started_at / ended_at.

### Phase Execution (`scripts/run_phase.sh`)

```bash
./evals/scripts/run_phase.sh <model> <output_subdir> [extra args...]
```

Examples:

```bash
./evals/scripts/run_phase.sh glm-5.1 glm-5.1-think-on
./evals/scripts/run_phase.sh glm-5.1 glm-5.1-think-off --no-think
./evals/scripts/run_phase.sh glm-5.1 _smoke --limit 5
```

Argument routing:
- `--no-vision` is forwarded to `igakuqa119` and `jmle2026` (both are vision tasks)
- All other flags (`--limit`, `--no-think`, `--max-tokens`, `--temperature`, `--base-url`) are forwarded to all tasks

Tasks executed: `llm-jp-eval-subset` 4 tasks + `igakuqa` + `igakuqa119` + `jmle2026` + `jmed-llm` 3 tasks = **10 tasks total**.

### Result Aggregation (`scripts/summarize.py`)

```bash
uv run --group evals python evals/scripts/summarize.py evals/results/<subdir>
# Diff against another run:
uv run --group evals python evals/scripts/summarize.py <a> --compare <b>
```

Output sections:
- **score table**: Per-task accuracy, `tok/s p50`, TTAT, think tokens
- **leaderboard rows**: Markdown rows ready to paste directly into official READMEs
- **timeline**: Per-task `started_at` / `ended_at` (ISO8601 + epoch_ms)

### Grafana Integration

Recorded in each task JSON:

```json
"started_at": "2026-04-28T14:44:07+09:00",
"ended_at": "2026-04-28T14:44:13+09:00",
"started_epoch_ms": 1777355047398,
"ended_epoch_ms": 1777355053624,
"duration_sec": 6.23
```

The `*_epoch_ms` values can be pasted into Grafana's Time Range (`from=...&to=...`) to retrospectively examine SGLang `:8000/metrics` (Prometheus) or DCGM exporter (GPU utilization/temperature) metrics for the corresponding time period.

## Supplement: Tasks Excluded from This Evaluation

The following 2 tasks from JMED-LLM were excluded in Phase 1. The same applies to Phase 2 onward (consistent across all phases). Excluding them **does not affect the relative comparison of the JMED-LLM MCQ 3 tasks**.

### SMDIS (excluded)

- **Scale**: 15,360 rows (an order of magnitude more than other tasks, 67% of all JMED-LLM)
- **Cost**: 13.7 s/q x 15,360 = **~55 hours per model** (~220h across all phases)
- **Data characteristics**: Determine from simulated SNS posts whether "the poster had disease X within 1 day" as an A/B classification. 87.5% are "B" (negative), **strongly imbalanced**
- **Relevance to clinical use**: SNS-style Japanese is a different domain from actual clinical records, radiology reports, and patient interviews
- **Decision**: Excluded on three grounds: excessive cost / weak imbalance / clinically irrelevant

### JCSTS (excluded)

- **Scale**: 3,670 rows / 19.5 s/q x 3,670 = **~20 hours per model** (~80h across all phases)
- **Data characteristics**: Semantic similarity judgment of medical sentences (6-level ordinal, linear-weighted kappa)
- **Rationale**:
  - Importance: low. Similarity judgment capability is indirectly measured by other MCQs
  - At 19% progress on GLM-5.1, accuracy was 0.349 (reference value; estimated to fall below the published LB GPT-4o score of 0.60 in kappa terms)
  - ~80h is not cost-effective for the sole purpose of "producing a JMED-LLM 4-task average"
- **Decision**: Can be individually run with `--task jcsts` if the task-specific value alone is desired

Both tasks are commented out in `evals/tasks/jmed_llm/run.py`'s `ALL_TASKS` list. **They are automatically skipped when using `--task all`**. Since they can still be run when explicitly specified, the plan is to collect results for all 3 models in a batch run when time permits.
